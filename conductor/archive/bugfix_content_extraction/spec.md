# Track Spec: 修复正文提取缺失问题

## 问题描述
用户反馈生成的 Markdown 文件中只有元数据和几个链接，缺少正文内容。

## 根本原因假设
1. `lib/parser.ts` 中的清洗逻辑可能过于激进，误删了正文内容。
2. `lib/converter.ts` 中的 Turndown 规则未能正确处理微信文章中深层嵌套的 `section` 标签，导致内容被忽略。
3. 微信文章的正文可能使用了非 `#js_content` 的加载方式（虽然不太可能，但需排除）。

## 修复方案
1. **增强 Debug 能力：** 在 parser 中添加日志，打印清洗前后的 HTML 长度和关键结构。
2. **调整 Parser 逻辑：** 检查 `#js_content` 下的子元素结构，避免由 `remove()` 调用误伤正文。
3. **优化 Converter 规则：** 确保 `section` 和 `span` 标签的内容能被正确透传，而不是被吞噬。特别要注意微信文章中大量使用 `span` 来控制样式，Turndown 默认可能会忽略行内元素的某些处理。
