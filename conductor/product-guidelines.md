# 产品准则

## 视觉与交互准则 (CLI)
- **现代感设计：** 使用丰富的 Emoji 表情（如 📦, 🚚, ✅, ❌）来标识任务状态。
- **色彩编码：** 
    - 蓝色：信息提示（Info）
    - 绿色：成功（Success）
    - 黄色：警告：警告（Warning）
    - 红色：错误（Error）
- **动态反馈：** 实现平滑的实时进度条，展示图片下载和处理的具体百分比。

## 内容处理准则
- **语义化转换：** 优先将微信 HTML 转换为标准的 Markdown 语法（如 `#`, `**`, `>`），而非复杂的 HTML 嵌套，确保在 Obsidian、VS Code 等工具中的完美兼容。
- **正文优先：** 自动识别并剔除文章末尾的干扰项，包括但不限于：
    - “往期推荐”列表
    - 公众号二维码图片
    - “点赞、在看、分享”动态图标组
    - 明显的底部横幅广告
- **鲁棒性：** 当单张图片下载失败时，不中断整个流程。在 Markdown 对应位置插入一个醒目的占位文本（例如：`![图片下载失败 - 错误日志查看输出](placeholder.png)`），并继续处理后续内容。

## 文件组织准则
- **原子性文件夹：** 确保每篇文章的所有资源（MD 文件、图片）都封装在自己的文件夹内，不与其他文章混杂。
- **资源路径：** 使用相对路径（如 `./assets/image.png`）引用图片，确保整个文件夹在移动或同步后依然能正确显示图片。
